## SECTION 1: MACROS

Macros allow you to record a sequence of commands and replay them multiple times.
This is incredibly powerful for repetitive editing tasks.

### Macro Basics

Recording a macro:
1. Press `q` followed by a letter (a-z) to start recording into that register
2. Perform your editing commands
3. Press `q` again to stop recording

Playing a macro:
- `@a` - play macro stored in register 'a'
- `@@` - repeat the last played macro
- `10@a` - play macro 'a' 10 times

## SECTION 2: REGISTERS

Registers are storage locations for text that you yank (copy) or delete.
Vim has 10 types of registers, but we'll focus on the most useful ones.

### Register Types

- `"` - Unnamed register (default for most operations)
- `0` - Yank register (last yank)
- `1-9` - Numbered registers (deletion history)
- `a-z` - Named registers (for macros and manual storage)
- `A-Z` - Append to named registers
- `+` - System clipboard
- `*` - Selection register (X11 primary selection)
- `:` - Command-line register
- `.` - Last inserted text
- `%` - Current filename
- `#` - Alternate filename

### Register Basics

- `"ayy` - Yank current line into register 'a'
- `"ap` - Paste from register 'a'
- `"Ayy` - Append current line to register 'a'
- `:reg` - Show all registers
- `:reg a` - Show register 'a'

### Advanced Register Techniques

#### Black Hole Register
- `"_dd` - Delete line without affecting registers

#### Expression Register
- `Ctrl+r =` - Evaluate expression

#### Register Concatenation
You can combine registers in insert mode:
- `Ctrl+r a` then `Ctrl+r b` pastes both

## SECTION 3: INDENTATION

Proper indentation is crucial for readable code.
Vim provides powerful commands for managing indentation.

### Indentation Basics

- `>>` - Indent line right (adds shiftwidth)
- `<<` - Indent line left (removes shiftwidth)
- `>` motion - Indent text (e.g., >ap indents paragraph)
- `<` motion - Unindent text
- `=` motion - Auto-indent text
- `==` - Auto-indent current line
- `=G` - Auto-indent to end of file

### Indentation Settings

- `:set shiftwidth=4` - Number of spaces for indent
- `:set tabstop=4` - Number of spaces a tab displays as
- `:set expandtab` - Use spaces instead of tabs
- `:set autoindent` - Copy indent from previous line
- `:set smartindent` - Smart indentation for programs
- `:set cindent` - C-style indentation

### Advanced Indentation Techniques

#### Indent Based on Syntax
- `=i{` - Indent inside braces
- `=a{` - Indent around braces (including braces)

## SECTION 4: TEXT OBJECTS

Text objects allow you to operate on specific text structures rather than lines or characters.
They make editing much more precise and efficient.

### Text Object Basics

Text objects have two forms:
- `i` - inside (excludes surrounding characters)
- `a` - around (includes surrounding characters)

Common text objects:
- `w` - word
- `W` - WORD (sequence of non-blank characters)
- `s` - sentence
- `p` - paragraph
- `(`, `)`, `b` - parentheses block
- `{`, `}` - braces block
- `[`, `]` - brackets block
- `<`, `>` - angle brackets block
- `"`, `'`, `` ` `` - quoted strings
- `t` - XML/HTML tag

### Text Object Examples

- `diw` - Delete inside word
- `daw` - Delete around word (including space)
- `ci(` - Change inside parentheses
- `ca(` - Change around parentheses
- `yi"` - Yank inside quotes
- `va"` - Visual select around quotes

### Advanced Text Object Techniques

#### Combining with Other Commands
- `d2iw` - Delete two words
- `c3i(` - Change inside three sets of parentheses

#### Custom Text Objects
You can create custom text objects with plugins or mappings:

```vim
" Map ii to select inside indentation
onoremap ii :<c-u>execute "normal! ?^\V\s*$\r:nohl\rkv"<CR>
```

#### Text Objects in Visual Mode
- Start visual mode, then use text object commands
- `viw` - Visual select inside word
- `va(` - Visual select around parentheses

### Exercise 1: Code Refactoring

Original code:

function processUser(user){return user.name+"-"+user.age;}
function getAddress(user){return user.street+", "+user.city;}
function formatDate(date){return date.getMonth()+1+"/"+date.getDate();}

Refactored code should be:

function processUser(user) {
  return user.name + " - " + user.age;
}

function getAddress(user) {
  return user.street + ", " + user.city;
}

function formatDate(date) {
  return (date.getMonth() + 1) + "/" + date.getDate();
}

Solution approach:
1. Record a macro to:
   - Add proper spacing around braces (f{ then a{)
   - Indent the return statements (>>)

2. Use registers to store formatting patterns
3. Apply to each function

2. Practice both 'inside' and 'around' versions
